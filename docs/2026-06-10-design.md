# aeye: standalone repo extraction + kitty zoom/pan

**Date:** 2026-06-10
**Status:** Design approved, pending spec review
**Origin:** Brainstormed out of lazytmux's image carousel (PR #42/#45).

## Summary

Extract lazytmux's Claude image carousel into a standalone, agent-agnostic
repo — `github.com/noamsto/aeye` — packaged like `tmux-state` (Go
binary + Nix flake + Home Manager module + its own Claude Code plugin
marketplace). Then add **zoom + pan** to the carousel for kitty-graphics
terminals.

The viewer is already agent-agnostic: it renders a per-pane manifest and has no
Claude knowledge. Only the *capture* (a Claude Code PostToolUse hook) is
Claude-specific. The extraction makes that split explicit — the manifest JSONL
is the stable interface, Claude is the first (and currently only) capture
adapter, and other agents can be added later as new adapters with zero viewer
changes.

## Goals

- A standalone `aeye` repo that builds green and that lazytmux
  consumes as a flake input — the carousel works exactly as it does today.
- Agent-agnostic core: viewer + manifest contract; Claude bits isolated under
  `adapters/claude-code/`.
- Zoom (`z`/`Z`) + panning in the carousel preview, on kitty/ghostty terminals.

## Non-Goals (explicit future work)

1. **D2 diagrams in the carousel.** A skill makes the agent emit `d2` fenced
   blocks; a Stop hook renders them `d2 → svg → resvg → png` (browser-free —
   never `d2 ... .png`, which shells to Playwright/Chromium) into the manifest.
   Deferred.
2. **Non-kitty zoom.** Zoom/pan for the `chafa` block-art fallback. Phase 1 is
   kitty-only; chafa zoom is a no-op.
3. **Other agent adapters** (Gemini CLI, Codex, aider, …). Structure leaves the
   door open; we build only the Claude adapter now.

## Architecture

### The manifest is the interface

Per-pane JSONL at `${CLAUDE_STATUS_DIR:-/tmp/claude-status}/images/<pane>.jsonl`,
one object per line:

```json
{"type":"image","path":"/abs/path.png","source":"Read","ts":"2026-...","mtime":1234567890}
```

- Keyed by tmux pane id (`%N` → `N`). Producer-agnostic: anything that appends
  valid lines shows up in the viewer.
- Consumers must tolerate duplicate `(path, mtime)` entries (concurrent hook
  firings) — already true today.
- Documented as `docs/MANIFEST.md` in the new repo — the contract every adapter
  targets.

The `/tmp/claude-status/images` path is **kept** (env-overridable) so there is
zero migration and the carousel coexists with lazytmux's claude-status machine
by subdirectory.

### Repo layout (`aeye`)

```
aeye/
  flake.nix                    # packages.default (binary), homeManagerModules.default, devShell, checks
  go.mod                       # module github.com/noamsto/aeye
  *.go                         # agent-agnostic viewer: gallery.go, gallery_render.go,
                               #   gallery_cache.go + zoom (+ tests)
  scripts/
    tmux-carousel-toggle.sh    # prefix+I toggle split (generic; targets $TMUX_PANE)
  adapters/claude-code/
    scripts/images-update.sh   # Claude Code PostToolUse capture hook
    claude-plugin/
      .claude-plugin/marketplace.json
      plugin manifest, hooks/hooks.json, skills/image-gallery/
  docs/MANIFEST.md
  CLAUDE.md  README.md
```

- **Binary** drops the `--gallery` multiplexing flag (that existed only to share
  one binary with the session/window pickers). It *is* the gallery:
  `aeye <pane>`; `main()` becomes `runGallery`.
- **Coupling stays loose:** reads the manifest, reads `@thm_*` tmux options
  (Catppuccin dark/light fallbacks are already built in, so it degrades without
  lazytmux), shells to `chafa`/`xdg-open`. No lazytmux Go dependency.
- **Flake/HM** mirror `tmux-state`: a Home Manager module exposes
  enable + options and wires the binary, hook, and plugin.

### The carve (changes in lazytmux)

- Delete `picker/gallery.go`, `picker/gallery_render.go`,
  `picker/gallery_cache.go` (+ their tests) and the 6-line `--gallery` dispatch
  block in `picker/main.go`.
- Delete `scripts/claude-images-update.sh`, `scripts/tmux-claude-images.sh`.
- Delete the `image-gallery` skill + image-capture hook entries from
  `claude-plugin/`.
- Add `aeye` as a flake input; repoint the `prefix + I` keybind and
  the gallery invocation in `config/tmux.conf.nix` at the new binary.
- Compose the new repo's HM module under `programs.lazytmux` (e.g.
  `programs.lazytmux.carousel.enable`, default on) and add its plugin to the
  plugin-dir list.

Separability was verified by compiling the gallery files in isolation: they
reference only gallery-local helpers (`clamp`, `digitKey`, `truncateToWidth`,
`termName`, their own `thmColor` method) **plus one straggler** — `detectTheme`
(a stdlib-only helper in `picker/main.go` that reads `theme-state.json`). With
`detectTheme` moved across, the gallery files compile and their tests pass
standalone. No other calls into the session/window picker code.

## Zoom / Pan (kitty)

### Approach

The kitty backend renders via the Unicode-placeholder protocol: an image is
transmitted store-only and a grid of placeholder cells (with row/column
diacritics) displays it scaled to a fixed cell box. **Zoom = crop the source in
Go and feed the existing downscale path; the placeholder fills the same cell
box, so a smaller crop reads as magnified.** No new kitty protocol surface.

### State (`galleryModel`)

- `curImg image.Image` + `curImgPath` — the decoded source of the *current
  selection*, decoded once on select/reload. Zoom/pan crops from this in-memory
  image (no per-keystroke re-decode, no disk churn).
- `zoom float64` — `1.0` = fit; clamped to a max (e.g. 8×, or until ~1 source
  pixel per cell).
- `panX, panY float64` — crop-center as a fraction (0..1), default `0.5/0.5`,
  clamped so the crop rectangle stays within the source bounds.

### Render

When `zoom > 1`, the preview transmit uses a crop rect
(`cropW = srcW/zoom`, `cropH = srcH/zoom`, centered on pan, clamped), scaled to
the preview cell box and encoded to a **fixed scratch PNG** (overwritten each
keystroke). Filmstrip thumbnails always stay fit. At `zoom == 1` the preview is
the full image (current behavior).

### Keys

- `z` / `+` / `=` — zoom in (`zoom *= 1.25`, clamp max).
- `Z` / `-` / `_` — zoom out (`zoom /= 1.25`, floor `1.0`; recenter pan at 1.0).
- `0` — reset zoom + pan.
- **Pan:** arrow keys *when `zoom > 1`*. Unzoomed, arrows still navigate
  (no behavior change). `h`/`l`/`j`/`k` always navigate. Pan step ≈ 10% of the
  visible crop.
- **Navigating to a new image resets zoom/pan to fit** (each image starts fit).
- Footer hint line shows pan keys while zoomed.

### chafa fallback

Zoom is a no-op on the chafa backend this phase (the "non-kitty zoom" future
item). Attempting it does nothing or flashes a hint.

## Known risks / refinements

Surfaced by adversarial review of the design (none fatal):

- **Zoom re-transmit must be preview-only.** Today's `transmitView()` does
  `deleteAll()` then re-transmits preview *and* filmstrip. Reusing it per
  pan keystroke = a full clear + retransmit every keypress (flicker, cost). Add
  a lighter path that deletes only `previewID` and re-places it; leave the
  filmstrip untouched while zooming.
- **The auto-refresh tick must not clobber zoom.** `galleryTickCmd` fires every
  1.5s and calls `reload()` → `transmitView()` + layout recompute. An image
  arriving mid-zoom would reset zoom/pan and drop the decoded `curImg`. Preserve
  zoom/pan across ticks; reset only on real navigation; re-decode `curImg` only
  when the *selected path* changes.
- **Namespace/env naming.** The agent-agnostic repo should not hardcode a
  Claude-named env. Use `AEYE_DIR` with a `CLAUDE_STATUS_DIR` fallback
  (and the same default root) — clean naming, zero migration, coexists with
  lazytmux's status machine by subdir.
- **Command/keybind rename churn.** Keep the `tmux-claude-images` command name
  (the skill text + `prefix + I` keybind reference it) — or rename across the
  skill, keybind, and toggle script atomically. Don't half-rename.

## Deployment note

aeye is not yet on GitHub; the input chain is
lazytmux → aeye → nix-config. **Order the carve so lazytmux never has
a broken intermediate:** wire the flake input + binary + plugin/skill first
(local `git+file://` until pushed), verify the carousel still works, *then*
delete the in-tree files — all in one branch. After merge, push aeye,
bump lazytmux's input to the GitHub ref, then bump the nix-config `lazytmux`
input — the same input-bump dance as prior carousel work.
